<task type="auto">
  <name>Task 1: Scaffold Next.js 16 project with tooling config</name>
  <files>
    package.json, pnpm-lock.yaml, tsconfig.json, next.config.ts, eslint.config.mjs,
    .prettierrc.json, postcss.config.mjs, vitest.config.mts, .gitignore
  </files>
  <action>
    **Step 1: Create Next.js 16 project**
    Run `pnpm create next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"` in the project root (the directory already exists as a git repo, so scaffolding must work in-place or use a temp dir and move files).

    If create-next-app refuses to scaffold in a non-empty directory, use a temp directory:
    ```
    pnpm create next-app@latest /tmp/manga-anime-tracker --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"
    ```
    Then copy the generated files into the project root, preserving existing `.planning/`, `.claude/`, and `.git/`.

    **Step 2: Install all dependencies**
    Core:
    ```bash
    pnpm add drizzle-orm @neondatabase/serverless pg next-themes dotenv
    ```
    Dev:
    ```bash
    pnpm add -D drizzle-kit @types/pg @eslint/eslintrc eslint eslint-config-next prettier @trivago/prettier-plugin-sort-imports prettier-plugin-tailwindcss vitest @vitejs/plugin-react
    ```

    **Step 3: Configure ESLint (project conventions)**
    Write `eslint.config.mjs` with flat config format:
    - Use `FlatCompat` for `next/core-web-vitals` and `next/typescript` extends
    - Rules: `@typescript-eslint/consistent-type-imports: error`, `no-console: [error, {allow: [warn, error]}]`, `@typescript-eslint/no-unused-vars` with `_` ignore patterns, `@typescript-eslint/explicit-function-return-type: error` for `.ts`/`.tsx` files
    - Ignores: `public`, `.next`, `next-env.d.ts`, `.docker`
    - See 01-RESEARCH.md "ESLint Flat Config" code example for exact content

    **Step 4: Configure Prettier**
    Write `.prettierrc.json`:
    - singleQuote: true, semi: false, printWidth: 90, arrowParens: "avoid"
    - Plugins: `@trivago/prettier-plugin-sort-imports`, `prettier-plugin-tailwindcss`
    - tailwindFunctions: ["cn", "cva"]
    - importOrder: ["^@/(.*)$", "^[./]"], importOrderSeparation: true, importOrderSortSpecifiers: true
    - See 01-RESEARCH.md "Prettier Config" code example for exact content

    **Step 5: Configure PostCSS**
    Write `postcss.config.mjs` with `@tailwindcss/postcss` plugin only (Tailwind v4 pattern).

    **Step 6: Update package.json scripts**
    Set scripts:
    - `"dev": "next dev"`
    - `"build": "next build"`
    - `"start": "next start"`
    - `"lint": "eslint ."` (NOT `next lint` -- removed in Next.js 16)
    - `"format": "prettier --write ."`
    - `"format:check": "prettier --check ."`
    - `"db:generate": "drizzle-kit generate"`
    - `"db:migrate": "drizzle-kit migrate"`
    - `"db:push": "drizzle-kit push"`
    - `"db:studio": "drizzle-kit studio"`
    - `"test": "vitest run --reporter=verbose"`

    **Step 7: Vitest config**
    Write `vitest.config.mts`:
    ```typescript
    import react from '@vitejs/plugin-react'
    import { resolve } from 'path'
    import { defineConfig } from 'vitest/config'

    export default defineConfig({
      plugins: [react()],
      resolve: {
        alias: {
          '@': resolve(__dirname, './src'),
        },
      },
      test: {
        environment: 'node',
        include: ['tests/**/*.test.ts'],
      },
    })
    ```

    **Step 8: Ensure .gitignore** includes: `.env.local`, `.env*.local`, `node_modules`, `.next`, `.docker/pgdata`.

    **Step 9: Run Prettier on all files** to ensure consistent formatting from the start.
    ```bash
    pnpm format
    ```

    **Step 10: Verify**
    - `pnpm build` must succeed
    - `pnpm lint` must pass with zero errors
  </action>
  <verify>
    <automated>pnpm build && pnpm lint && pnpm format:check</automated>
  </verify>
  <done>
    Next.js 16 project builds successfully. ESLint passes with project convention rules. Prettier formatting is consistent. All config files (eslint, prettier, postcss, vitest) are in place.
  </done>
</task>

<task type="auto">
  <name>Task 2: Set up Docker DX with Makefile and environment files</name>
  <files>
    Makefile, .docker/Makefile, .docker/docker-compose.yaml,
    .docker/make/01_setup.mk, .docker/make/02_docker.mk,
    .env.example, .env.local
  </files>
  <action>
    **Step 1: Docker Compose setup**
    Create `.docker/docker-compose.yaml`:
    - PostgreSQL 17-alpine only (no mailer service)
    - Port mapping `5433:5432` (avoid host PostgreSQL conflict per pitfall #6)
    - Environment vars: POSTGRES_USER=manga_tracker, POSTGRES_PASSWORD=manga_tracker, POSTGRES_DB=manga_tracker
    - Healthcheck: `pg_isready -U manga_tracker`
    - Named volume `pgdata`

    **Step 2: Makefile structure (Docker include pattern)**
    Create `.docker/Makefile`:
    - Wildcard include of `.docker/make/*.mk`
    - `default: setup up` target
    - `help` target

    Create `.docker/make/01_setup.mk`:
    - `setup` target ensuring `node_modules` directory exists

    Create `.docker/make/02_docker.mk`:
    - `DOCKER_COMPOSE` variable pointing to `.docker/docker-compose.yaml`
    - Targets: `up` (with `--wait`), `halt` (stop), `destroy` (down --remove-orphans --volumes), `ps`, `logs`
    - No prod/ENV logic (single environment)

    Create root `Makefile`:
    - `PROJECT_NAME` derived from directory name
    - `include .docker/Makefile`

    **Step 3: Environment files**
    Create `.env.example`:
    ```
    # Local development (Docker PostgreSQL)
    LOCAL_DATABASE_URL=postgresql://manga_tracker:manga_tracker@localhost:5433/manga_tracker

    # Production (Neon PostgreSQL) -- set in Vercel dashboard
    # NEON_DATABASE_URL=

    # Node environment
    NODE_ENV=development
    ```

    Create `.env.local` (gitignored, actual local values):
    ```
    LOCAL_DATABASE_URL=postgresql://manga_tracker:manga_tracker@localhost:5433/manga_tracker
    NODE_ENV=development
    ```

    **Step 4: Smoke test Docker**
    - `make up` must start PostgreSQL container (verify with `docker compose ps` or `.docker/docker-compose.yaml` status)
    - `make halt` must stop it cleanly
  </action>
  <verify>
    <automated>make up && make halt</automated>
  </verify>
  <done>
    Docker PostgreSQL starts via `make up` on port 5433 and stops via `make halt`. Makefile follows include pattern structure. Environment files (.env.example, .env.local) contain correct connection strings.
  </done>
</task>
