  <action>
    1. Rewrite `src/app/(app)/page.tsx` (server component):
       - Import `auth`, `headers`, `getTrackingList`, `getStatusCounts`, `FilterBar`, `TrackingGrid`, `EmptyState`
       - Accept `searchParams` prop: `{ searchParams: Promise<{ status?: string, type?: string, sort?: string }> }`
       - Get session (already guaranteed by layout.tsx auth check)
       - Parse and validate URL params: status must be one of the enum values or undefined, mediaType must be 'anime'|'manga' or undefined, sort must be 'rating'|'title'|'date_added' or undefined (default 'date_added')
       - Fetch in parallel: `Promise.all([getTrackingList({ userId, status, mediaType, sort, page: 1 }), getStatusCounts(userId)])`
       - Layout structure:
         ```
         <main className="mx-auto max-w-7xl">
           <FilterBar statusCounts={counts} />
           {totalEntries === 0 && no active filters ? <EmptyState title="Your collection is empty" description="Start tracking anime and manga from the search page" actionLabel="Browse & Search" actionHref="/search" />
           : entries.length === 0 && has active filters ? <EmptyState title="No results" description={`No ${mediaType ?? ''} entries with ${status ?? 'any'} status`} (with reset filters button) />
           : <TrackingGrid key={`${status}-${mediaType}-${sort}`} initialEntries={entries} initialHasMore={hasMore} currentFilters={{ status, mediaType, sort }} /> }
         </main>
         ```
       - CRITICAL: Pass `key={status + '-' + mediaType + '-' + sort}` to TrackingGrid so React remounts it on filter/sort change, resetting infinite scroll state (per research pitfall #2)
       - Show FilterBar always (even on empty collection so user can see the zero counts)

    2. Create `src/app/(app)/tracking-grid.tsx` (client component with 'use client'):
       - Props: `{ initialEntries: TrackingEntry[], initialHasMore: boolean, currentFilters: { status?, mediaType?, sort? } }`
       - State: `entries` (initialized from initialEntries), `hasMore` (from initialHasMore), `page` (starts at 1)
       - Uses `useTransition` for non-blocking page loads (established pattern)
       - IntersectionObserver sentinel pattern (native, no library):
         - `sentinelRef = useRef<HTMLDivElement>(null)`
         - `useEffect` creates IntersectionObserver watching sentinel div
         - On intersection (threshold 0.1) AND !isPending AND hasMore: call `fetchTrackingPage({ ...currentFilters, page: page + 1 })` inside `startTransition`
         - On result: append entries, update hasMore, increment page
         - Include `page`, `hasMore`, `isPending`, and `currentFilters` in useEffect dependency array to avoid stale closure (per research pitfall #1)
         - Disconnect and reconnect observer when deps change
       - Grid layout: same responsive classes as PlaceholderGrid: `grid grid-cols-2 gap-4 p-4 md:grid-cols-3 lg:grid-cols-4 xl:grid-cols-5 2xl:grid-cols-6`
       - Render: `entries.map(e => <TrackingCard key={e.id} entry={e} />)`
       - Loading state: when isPending, show 4 SkeletonCards at the end of the grid
       - Sentinel div: `<div ref={sentinelRef} className="col-span-full h-4" />` at end of grid (invisible trigger zone)
  </action>
