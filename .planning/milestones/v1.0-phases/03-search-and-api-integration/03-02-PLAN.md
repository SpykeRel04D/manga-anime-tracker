      - If no session, return `{ success: false, error: 'not_authenticated' }`
      - Call `addTrackingEntry(session.user.id, data)` and return result
    - Export `async function getUserTrackedIds(): Promise<number[]>`:
      - Get session, if none return []
      - Query `db.select({ anilistId: trackingEntries.anilistId }).from(trackingEntries).where(eq(trackingEntries.userId, session.user.id))`
      - Return array of anilistIds

    - Async function reading `searchParams` (await props.searchParams per Next.js 16 pattern)

    - If no `q`, pass empty results
    - Render: `<main>` with `max-w-3xl mx-auto px-4 py-6` container
      - `<h1>` "Search" with `text-2xl font-bold mb-4`
      - `<Suspense>` wrapping `<SearchInput />`
      - Conditionally render: if query exists and results loading, show `<SearchSkeleton />`; otherwise `<SearchResults results={results} query={q} trackedIds={trackedIds} />`
    - Use Suspense boundary around SearchInput since it uses useSearchParams (per Next.js docs)

    - `'use client'`
    - Follow exact Next.js search pattern from RESEARCH.md:
      - `useSearchParams`, `usePathname`, `useRouter` from `next/navigation`
      - `useDebouncedCallback` from `use-debounce` with 300ms delay
      - `handleSearch` callback: creates URLSearchParams, sets or deletes 'q', calls `replace(pathname?params)`
    - Render: Full-width input with search icon (Lucide `Search`), `defaultValue` from `searchParams.get('q')`, placeholder "Search anime or manga...", styled with shadcn Input component + `pl-10` for icon padding
    - Input wrapped in relative div with absolute-positioned Search icon on the left

    - Renders 5 horizontal skeleton cards matching the search result card layout
    - Each skeleton: flex row with a `Skeleton` rectangle (80px wide, aspect-[3/4]) on left for cover image, and right side with `Skeleton` lines for title (h-5 w-3/4), subtitle (h-3 w-1/2), badges (h-5 w-20), and episode count (h-4 w-24)
    - Reuse shimmer pattern from existing `skeleton-card.tsx` by using the same `Skeleton` component

    **Step 7: Create `src/app/(app)/search/search-result-card.tsx` (Client Component)**
    - `'use client'`
    - Props: `result: MediaSearchResult`, `isTracked: boolean`, `trackedStatus?: string`
    - Per locked decisions: horizontal card layout, cover image (~80-100px) on left, text info on right
    - Layout: `flex gap-4` row inside a Card component with `overflow-hidden`
    - Cover image: `next/image` with `width={80} height={113}` (3:4 aspect), `alt={result.title}`, fallback to placeholder div with gradient if coverImageUrl is null
    - Right side content:
      - Title (English primary, larger): `result.title` in `font-semibold text-sm`
      - Subtitle (Romaji secondary, smaller): only show `result.titleRomaji` as secondary if `result.titleEnglish` exists and differs from romaji, in `text-xs text-muted-foreground`
      - Type badge: `<Badge>` with variant -- "ANIME" uses `className="bg-chart-3/20 text-chart-3 border-chart-3/30"` (blue), "MANGA" uses `className="bg-chart-2/20 text-chart-2 border-chart-2/30"` (green). Text: result.type. Use `variant="outline"` as base.
      - Status text: human-readable status (map FINISHED -> "Finished", RELEASING -> "Airing", NOT_YET_RELEASED -> "Upcoming", CANCELLED -> "Cancelled", HIATUS -> "On Hiatus"), in `text-xs text-muted-foreground`
      - Episode/chapter count: Show "X episodes" for anime (if episodes not null), "X chapters" for manga (if chapters not null), in `text-xs text-muted-foreground`
    - Action area (right side, bottom or flex end):
      - If `isTracked`: show status Badge (e.g., "Watching", "Plan to Watch") in `variant="secondary"` -- non-interactive, provides context
      - If not tracked: "Add to list" Button (size="sm", variant="secondary") with Plus icon. On click: call `addToTrackingList` server action via `useTransition`, show toast on success/error, use `useRouter().refresh()` to update tracked state
    - Cards are NOT clickable as a whole per locked decision. Only the Add button is interactive.

    - Props: `results: MediaSearchResult[]`, `query: string`, `trackedIds: number[]`

    - If `query` exists but `results` is empty: render "No results found for '[query]'" text message with muted styling. Include a subtle "Check your spelling or try a different term" suggestion underneath.
    - Otherwise: render results as a vertical list (`flex flex-col gap-3`) of `<SearchResultCard>` components, passing `isTracked={trackedIds.includes(result.id)}`
