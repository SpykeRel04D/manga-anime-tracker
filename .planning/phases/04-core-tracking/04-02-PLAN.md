    - `changeProgress(entryId: string, progress: number)` -> calls `updateProgress`
    - `changeRating(entryId: string, rating: number | null)` -> calls `updateRating`

    - Await `params` (Next.js 16: `const { id } = await params`)
    - Get session, redirect to `/login` if not authenticated
    - Call `getTrackingEntry(session.user.id, id)`, show `notFound()` if null
    - Render `<TrackingEditForm entry={entry} />`

    **Step 4: Star rating component** -- Create `src/app/(app)/tracking/[id]/star-rating.tsx`:
    - `'use client'` component
    - Props: `value: number | null`, `onChange: (value: number | null) => void`, `disabled?: boolean`
    - Renders 10 Star icons (lucide-react) in a row
    - State: `hovered: number | null` for hover preview
    - On hover star N: fill stars 1-N in lighter shade (`amber-400`)
    - Filled stars (value set): `fill-amber-500 text-amber-500`
    - Empty stars: `fill-transparent text-muted-foreground/40`
    - Click star N: if value === N, call onChange(null) (toggle off); else call onChange(N)
    - Show "N/10" text next to stars when value is not null
    - Per user decision: 10 amber/gold stars, hover preview lighter shade, click same star removes rating

    **Step 5: Progress stepper component** -- Create `src/app/(app)/tracking/[id]/progress-stepper.tsx`:
    - `'use client'` component
    - Props: `value: number`, `total: number | null`, `label: 'eps' | 'ch'`, `onChange: (value: number) => void`, `disabled?: boolean`
    - Layout: Minus button | clickable number display | Plus button
    - Minus button (lucide Minus): calls onChange(value - 1), disabled when value <= 0
    - Plus button (lucide Plus): calls onChange(value + 1), disabled when total is known and value >= total
    - Clickable number: clicking the number turns it into an Input for direct entry; on blur or Enter, calls onChange with the typed value (clamped client-side to 0..total)
    - Display format: "3 / 12 eps" or "45 / ? ch" (when total is null, show "?")
    - Per user decision: stepper + direct input combo, clamped to total, "eps" for anime / "ch" for manga

    **Step 6: Tracking edit form** -- Create `src/app/(app)/tracking/[id]/tracking-edit-form.tsx`:
    - `'use client'` component accepting `entry: TrackingEntry` prop
    - Local state for each field: status, progress, rating, notes (optimistic updates)
    - Each field change uses its own `useTransition` for independent auto-save
    - **Layout** (per user decision): Cover image + titles at top, then vertically stacked: status dropdown, progress stepper, star rating, notes textarea, remove button
    - **Cover image**: Next.js Image component, large display at top, rounded corners
    - **Titles**: English title as h1, Romaji subtitle if different
    - **Status dropdown**: shadcn Select with 5 options (Watching, Completed, On Hold, Dropped, Plan to Watch). On change: set local state immediately, call `changeStatus` server action, toast "Status updated" on success, rollback on failure
    - **Progress stepper**: ProgressStepper component. `total` = `entry.totalEpisodes` for anime, `entry.totalChapters` for manga. `label` = 'eps' for anime, 'ch' for manga. On change: set local state, call `changeProgress`, if `result.statusChanged` show toast "Marked as Completed!" and update local status to 'completed', otherwise toast "Progress updated"
    - **Star rating**: StarRating component. On change: set local state, call `changeRating`, toast "Rating updated" or "Rating removed" based on value
    - **Notes textarea**: shadcn Textarea with auto-resize via onInput height adjustment (see RESEARCH.md fallback pattern). Placeholder "Add personal notes...". On blur: if value changed from initial, call `changeNotes`, toast "Notes saved"
    - **Remove button**: Button variant="destructive" with Trash2 icon. Wrapped in shadcn AlertDialog: title "Remove from your list?", description "Remove [title] from your list? This cannot be undone.", Cancel and Remove actions. On confirm: call `removeFromList`, redirect to `/search` via `router.push('/search')`
    - **Metadata refresh**: On mount, call `refreshEntryMetadata(entry.id)` (no force). If result.refreshed === true, call `router.refresh()` to reload server component with fresh data. Also render a RefreshCw icon button that calls `refreshEntryMetadata(entry.id, true)` (force=true) and refreshes page
    - **Error handling**: All server action failures show `toast.error` with appropriate message and rollback local state to previous value

    **Step 7: Enhance search result card** -- Modify `src/app/(app)/search/search-result-card.tsx`:
    - When `isTracked` is true: the status badge should be a Link to `/tracking/[entryId]` instead of static Badge
    - Need to change props: pass `trackedEntryId: string | null` instead of `isTracked: boolean` so the card can build the link URL
    - Update the already-tracked section to render: `<Link href={/tracking/${trackedEntryId}}><Badge variant="secondary">Plan to Watch</Badge></Link>`
    - Also update the toast on successful add: `toast.success(...)` should include a link/action. Use Sonner's action option: `toast.success('Added to your list', { action: { label: 'Manage', onClick: () => router.push(/tracking/${newEntryId}) } })`. This requires the `addToTrackingList` action to return the new entry ID.

    **Step 8: Update search actions** -- Modify `src/app/(app)/search/actions.ts`:
    - Change `getUserTrackedIds` to return `Array<{ anilistId: number; entryId: string }>` instead of just `number[]` so search cards can link to edit pages
    - Update `addToTrackingEntry` use case to return the new entry ID on success (modify the insert to use `.returning({ id: trackingEntries.id })` and return it)
    - Update `addToTrackingList` server action return type to include `entryId` on success
